1、iframe

1、<iframe src="https://www.baidu.com" name="xxx"></iframe>嵌套页面

2、<iframe name="xxx" src="#" frameborder="0"></iframe> name这个属性与a标签结合在一起才有用
<a href="http://qq.com" target="xxx">qq</a>
点击qq，qq页面会在iframe里出现

3、在index.html里写:
<iframe src="./index2" frameborder="0"></iframe>
index2.html里写:
<iframe src="./index3" frameborder="0"></iframe>
index3.html里写:
<a href="http://qq.com" target=_blank>qq</a>（点击第一个qq会在一个新的标签页面里打开）
<a href="http://qq.com" target=_self>qq</a>（点击第二个qq会在index3的嵌套页面里打开）
<a href="http://qq.com" target=_parent>qq</a>（点击第三个qq会在index2的嵌套页面里打开）
<a href="http://qq.com" target=_top>qq</a>（点击第四个qq会在index里打开）

2. a

中页面是查看还是下载Content-type：octet-stream或者用a标签强制下载，<a href="http://qq.com" download>下载</a>

<a href="//qq.com">QQ</a>href里的是无协议地址，当前用的是http协议就自动继承http协议，用的是file就继承file协议。
为了解决，在Git Bash 运行npm i -g http-server可安装一个服务器，在index.html的文件夹运行http-server可开启服务器，在浏览器浏览终端返回的地址即可用http协议来访问写的文件。

a.当在index.html中点击<a href="xxx.html">qq</a>，会访问/xxx.html而不是/index.html/xxx.html
b.<a href="?name=zy">qq</a>点击之后会在当前页面发起?name=zy的请求
c.<a href="#sss">qq</a>点击之后不会发起请求，因为锚点作用是在页面内的跳转
d.<a href="/..">link</a>标签被点击后浏览器会发起GET / HTTP/1.1的请求，路径还是/，因为/已经是根目录，无法再往前退

伪协议:
a.<a href="javascript:;">qq</a>点击之后不会做任何事情
b.<a href="javascript:alert(1)";>qq</a>点击之后会弹出1

         <li> `</form>`</li>                  

    </ol>
